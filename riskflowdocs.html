Exchange Supportati

RiskFlow si connette direttamente alle API ufficiali di ogni exchange tramite un proxy sicuro. Ogni exchange è configurato in modo separato e indipendente — passare da uno all'altro ricarica automaticamente chart e dati.

Bitget

  • Ordini Market / Limit
  • Ladder Orders
  • SL / TP inline
  • Key + Secret + Passphrase
Bybit
Supportato

  • Ordini Market / Limit
  • SL / TP separati
  • Modifica SL/TP live

Bybit Demo
Demo
  • Ambiente di test ufficiale
  • Stesse API di Bybit Live
  • Fondi virtuali
  • Perfetto per testare strategie

ℹ

Configurazione API

1
Apri il pannello API
Clicca sul pulsante exchange in alto a destra nella topbar (es. "Bitget"). Si apre il modal di configurazione con i tab per ogni exchange.
2
Seleziona l'exchange
Clicca sul tab dell'exchange che vuoi configurare. I form sono separati e indipendenti — compilarne uno non influenza gli altri.
3
Inserisci le credenziali
Compila i campi richiesti. Solo Bitget richiede anche la Passphrase. Bybit e Bybit Demo richiedono solo API Key e Secret.
4
Salva e connetti
Clicca "Salva". Le chiavi vengono cifrate con AES-GCM e salvate su Firestore. La chart si ricarica automaticamente con i dati dell'exchange selezionato.

03

Sicurezza

🔐
AES-GCM Encryption
Le API key vengono cifrate lato client con AES-256-GCM prima di essere salvate. La chiave di cifratura è derivata dall'UID utente — nessuno oltre a te può decifrarle.
☁️
Firestore Storage
Le chiavi cifrate sono sincronizzate su Firebase Firestore, protette dalle regole di sicurezza Firebase. Accessibili solo con il tuo account.
🔀
Proxy Sicuro
Tutte le chiamate agli exchange passano per un proxy server che firma le richieste server-side. Le chiavi non vengono mai esposte nel browser network.
🛡️
Masking UI
Dopo il salvataggio, le chiavi vengono mostrate mascherate (••••••••). Il form distingue tra "chiave invariata" e "nuova chiave inserita".

SL

TP1

TP2
Take Profit 2. Secondo livello target con R:R separato.
TP3

ℹ

Come usare le linee — workflow tipico

1
Identifica la zona di ingresso
Individua sulla chart il livello dove vuoi entrare (es. supporto, resistenza, pattern). Per ordini Market la linea ENTRY segue il prezzo live automaticamente — non serve impostarla. Per ordini Limit, trascina la linea gialla nel punto desiderato.
2
Posiziona lo Stop Loss
Trascina la linea rossa SL sotto il minimo della struttura (long) o sopra il massimo (short). Oppure: tasto destro sulla chart → Set Stop Loss. Non appena la linea è impostata, la size in USDT e il margine richiesto vengono calcolati in tempo reale nel pannello.
3
Imposta i Take Profit
Attiva il toggle Take Profits nel pannello, poi trascina TP1, TP2, TP3 nei livelli target. Ogni linea mostra subito il R:R nel badge (es. R:R 1:2.4). Abilita solo i TP che intendi usare con i checkbox.
4
Regola fine con drag preciso
Avvicina il cursore alla linea finché compare la maniglia ⇅ SL ⇅ — poi trascina. Per rimuovere una linea clicca il cerchietto ✕ che appare sopra la maniglia. Il prezzo aggiornato appare in tempo reale nel badge a destra.

Calcolo Rischio & Size

RiskFlow calcola automaticamente la size dell'ordine in base al rischio definito, al prezzo di entry e allo stop loss. Il calcolo aggiorna in tempo reale ad ogni modifica.

Parametro Tipo Descrizione
Balance USDT Capitale totale del conto. Inseribile manualmente o sincronizzato dall'exchange.
Risk % % Percentuale del balance da rischiare per questo trade.
Risk $ USDT Ammontare fisso in USDT da rischiare. Alternativo al Risk %.
Leverage 1x–125x Leva applicata. Influenza il margine richiesto, non la size calcolata.
Margin Mode Crossed / Isolated Modalità margine. Crossed = tutto il conto come garanzia. Isolated = solo il margine allocato.
Formula size 
// Calcolo size in contratti
const riskUsd    = balance × (risk / 100);
const distSL      = Math.abs(entry - sl) / entry; // % distanza SL
const sizeUsdt    = riskUsd / distSL;
const contracts   = sizeUsdt / entryPrice;

// Arrotondamento al step size dell'exchange
const finalContr  = Math.floor(contracts / stepSize) × stepSize;

Esempio pratico — BTC Long

Scenario: BTCUSDT Long a mercato
Input
Balance$10,000 USDT
Risk %1%
Entry (market)$84,500
Stop Loss$83,100
Leverage10x
Risultato calcolato
Risk $$100 USDT
Dist. SL1.66%
Size$6,024 USDT
Contratti0.0713 BTC
Margine req.$602 USDT
Calcolo passo passo
// 1. Rischio in USDT
riskUsd  = 10000 × (1 / 100) = $100

// 2. Distanza SL %
distSL   = |84500 - 83100| / 84500 = 1.657%

// 3. Size in USDT (quanta posizione aprire)
sizeUsdt = 100 / 0.01657 = $6,036 USDT

// 4. Margine richiesto (con leva 10x)
margin   = 6036 / 10 = $603 USDT

// → Se BTC va a $83,100, perdi esattamente $100 = 1% del balance ✓

Risk % vs Risk $ — quando usare quale

📊
Risk % — consigliato
Il rischio si adatta automaticamente al balance. Se il conto cresce da $10,000 a $15,000, con Risk 1% rischi $150 invece di $100 — la size scala proporzionalmente.

Ideale per: gestione professionale del conto, crescita composta, rispetto delle regole di money management.
💵
Risk $ — importo fisso
Rischi sempre lo stesso importo in USDT indipendentemente dal balance. Se imposti $50, rischi sempre $50 per trade.

Ideale per: test su Bybit Demo, trading con regole fisse, situazioni dove vuoi controllare esattamente il capitale esposto per trade.
⚠
La leva non cambia il rischio — cambia solo il margine richiesto. Con 1% di risk e leva 10x rischi comunque $100, ma il margine bloccato è $603 invece di $6,036. Aumentare la leva non aumenta il guadagno potenziale calcolato — aumenta solo il rischio di liquidazione.

Stop Loss & Take Profit

SL e TP possono essere impostati graficamente sulla chart o inseriti manualmente nei campi numerici. Il sistema valida sempre che SL sia nella direzione corretta rispetto all'entry.

🛑
Stop Loss
Obbligatorio per il calcolo della size. Impostabile via:

• Drag della linea rossa sulla chart
• Click su "Set SL" poi click sulla chart
• Input numerico nel pannello

La size si aggiorna in tempo reale.
🎯
Take Profit (fino a 3)
Fino a 3 livelli TP abilitabili indipendentemente. Ogni TP mostra il R:R calcolato automaticamente.

• TP1 — verde
• TP2 — ciano chiaro
• TP3 — azzurro

Ogni TP ha un checkbox on/off.

Modifica SL/TP su posizioni aperte

Nel pannello posizioni aperte puoi modificare SL e TP direttamente sulle posizioni già aperte, senza chiuderle. Sono disponibili anche:

⚖️
Breakeven (BE)
Sposta automaticamente lo SL al prezzo di entry della posizione, portando il rischio a zero.

Quando usarlo: dopo che il prezzo ha raggiunto TP1 e vuoi proteggere i guadagni. Clicca il pulsante BE sulla linea entry nella chart oppure nel pannello posizioni.
📉
Chiusura Parziale
Chiudi una percentuale della posizione (25%, 50%, 75%, 100%) con slider interattivo.

Quando usarlo: per ridurre l'esposizione su una resistenza importante senza chiudere tutta la posizione.
ℹ
Le linee SL/TP sulle posizioni aperte sono draggabili direttamente dalla chart — le vedi sovrapposte al grafico in stile TradingView. Trascina la linea SL o TP al nuovo livello, poi premi Set nel popup per inviare la modifica all'exchange.

Ladder Orders

La modalità Ladder permette di distribuire l'ingresso su 4 livelli di prezzo diversi (DCA). Ogni livello ha un prezzo impostabile e una percentuale della size totale.

1
Seleziona "Ladder" nel tipo ordine
Il pannello laterale mostra 4 slot (L1–L4) con prezzo e percentuale configurabile per ognuno.
2
Imposta i livelli
Trascina le 4 linee ladder sulla chart (LD1–LD4) oppure usa il click "Set L1/L2/L3/L4". Le percentuali devono sommare al 100%.
3
Imposta SL e TP
SL e TP sono condivisi e si applicano all'intera posizione ladder. L'entry price medio viene calcolato automaticamente.
4
Invia tutti gli ordini
Con un singolo click vengono inviati tutti i livelli abilitati come ordini Limit separati sull'exchange.
⚠
La somma delle percentuali dei livelli deve essere esattamente 100% prima di poter inviare il ladder. Il tool mostra un avviso se la somma è errata.

Esempio pratico — DCA su ETH

ETHUSDT Long ladder — Balance $10,000 · Risk 2% · SL $1,800 · Leva 5x
LivelloPrezzoPeso %Size USDTMargineSL dist.
L1 $2,200 15% $750 $150 18.2%
L2 $2,100 25% $1,153 $231 14.3%
L3 $2,000 30% $1,500 $300 10.0%
L4 $1,920 30% $1,667 $333 6.3%
Totale / Media 100% $5,070 USDT $1,014 Entry medio: ~$2,020
✓
Distribuendo l'ingresso su 4 livelli il prezzo medio di entry si abbassa rispetto a comprare tutto a $2,200. Se ETH scende a $1,920 prima di salire, il DCA ladder porta l'entry medio a ~$2,020 invece di $2,200 — riducendo la distanza dallo SL e migliorando il R:R finale.

Distribuzione pesi — automatica vs manuale

🤖
Distribuzione automatica
Di default RiskFlow distribuisce i pesi in modo crescente: L1 fisso al 15%, gli altri livelli aumentano progressivamente verso lo SL (dove è più probabile che il prezzo vada).

Logica: più il livello è vicino allo SL, maggiore è il rischio che venga toccato → più peso = posizione media migliore se il prezzo scende fin lì.
✏️
Pesi manuali
Puoi sovrascrivere il peso di ogni livello direttamente nel campo % accanto al prezzo. Il tool verifica che la somma sia 100% prima di consentire l'invio.

Esempio: vuoi pesare di più il primo livello se pensi che il prezzo salga subito → L1 50%, L2 30%, L3 15%, L4 5%.

PnL Realtime

Il PnL di ogni posizione viene calcolato localmente ogni 1.5 secondi usando il prezzo mark aggiornato, senza ricaricare tutta la dashboard dall'exchange.

💹
Price Cache
I prezzi mark vengono cachati per simbolo. Il ticker del simbolo attivo sulla chart aggiorna la cache ogni 1.5s. Per le altre posizioni un background fetcher separato aggiorna ogni 2s.
🔢
Formula PnL
Long: (mark − entry) × size
Short: (entry − mark) × size

ROE = PnL / Margin × 100
ℹ
Il prezzo mark viene aggiornato ogni 1.5s per il pair attivo sulla chart, e ogni 2s per gli altri pair con posizioni aperte tramite un background fetcher separato.

Troubleshooting

⚠️ PnL mostra 0 o non si aggiorna
Il PnL viene calcolato localmente usando la price cache. Se risulta 0:

• Verifica che il ticker del pair sia attivo sulla chart
• Controlla che le API key abbiano il permesso Read abilitato
• Premi il pulsante Refresh nel pannello posizioni per forzare il reload
ℹ️ Chart non cambia al cambio exchange
La chart si ricarica automaticamente quando si switcha il tab exchange nel modal API. Se non succede, clicca manualmente su un altro exchange e poi torna a quello desiderato. Ogni exchange ha il suo fetcher dedicato con fallback su Bitget.
🔐 Errore autenticazione API
• Verifica che le chiavi abbiano i permessi Trade abilitati
• Per Bitget: la Passphrase è obbligatoria
• Controlla che l'IP del proxy sia whitelistato nelle impostazioni API dell'exchange
• Le chiavi mascherate (••••) nel form indicano che le credenziali sono già salvate
✓ La size calcolata è troppo piccola
Se la size risulta inferiore alla minTradeNum dell'exchange, l'ordine viene bloccato con un avviso. Soluzioni:

• Aumenta il Risk % o l'importo fisso
• Avvicina lo SL all'entry (distanza minore = size maggiore)
• Usa un simbolo con step size minore
ℹ️ Le linee sulla chart scompaiono al cambio TF
Le linee Entry, SL e TP non vengono resettate al cambio timeframe — i prezzi rimangono gli stessi. Se le linee sembrano sparite è perché sono fuori dal range visibile della chart. Usa il pulsante ⊙ live per tornare al prezzo attuale e rivedere le linee.
🔮 Il calcolo della size non si aggiorna
Il calcolo richiede tutti e 3 i valori: Risk, Entry (o prezzo live in modalità Market) e SL. Se uno dei tre è mancante o 0, il risultato mostra "—".

• In modalità Market: assicurati che il ticker live sia attivo (il punto verde lampeggia nella topbar)
• In modalità Limit: inserisci il prezzo di entry nel campo apposito
• Verifica che il campo Risk non sia vuoto o 0
❌ Ordine rifiutato dall'exchange
Cause comuni e soluzioni:

• Insufficient margin: il margine disponibile è inferiore a quello richiesto — riduci la size o abbassa la leva
• Price too far from mark: su Bybit il prezzo limite deve essere entro certi range dal mark price — avvicina l'entry
• Invalid qty: la size non rispetta lo step size dell'exchange — RiskFlow arrotonda automaticamente, ma verifica la leva e il balance
• Position mode: Bybit richiede che la modalità posizione (One-Way vs Hedge) sia configurata correttamente nelle impostazioni dell'exchange
⚠️ Bybit Demo — errore connessione
L'ambiente Demo di Bybit usa endpoint separati dal live. Se la connessione fallisce:

• Verifica di aver creato le chiavi dalla sezione Demo Trading → API Management (non dal live)
• Le chiavi Demo scadono ogni 3 mesi se non associate a un IP — rinnova dalla pagina API Management del Demo
• Se il balance mostra "—" anche dopo il salvataggio, premi ⟳ Sync per forzare il refetch
